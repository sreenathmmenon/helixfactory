HelixFactory Twin Gap Assessment vs Top Competitors
Generated: 2026-05-27

Purpose
-------
This document compares the current HelixFactory Twin feature against five
competitors/reference products:

1. Sourcegraph
2. GitNexus
3. CodeSee
4. CodeLogic
5. SciTools Understand

The goal is to identify exactly where HelixFactory is lacking and what must be
done to make Twin feel enterprise-grade, smooth, visually strong, and useful
enough for companies to buy.

Executive Truth
---------------
HelixFactory Twin is not yet better than these competitors. It has a promising
foundation, but the current Twin feature is still closer to an early product
prototype than a world-class enterprise graph/code intelligence experience.

It is not fair to say it has reached only 1% of every competitor, because the
foundation already includes real ingestion, graph traversal, centered subgraph
queries, risk overlay, pre-mortem linkage, BYOK AI boundary, and audit-oriented
architecture. Most competitors do not combine all of those in one workflow.

However, if judged as a polished enterprise product that a buyer can adopt
without hesitation, the current Twin is still far behind:

- Compared to Sourcegraph search/navigation depth: about 10-15%.
- Compared to GitNexus graph-native exploration: about 20-25%.
- Compared to CodeSee map/review/onboarding UX: about 10-15%.
- Compared to CodeLogic impact/dependency intelligence: about 10-15%.
- Compared to Understand static analysis depth/metrics/graph modes: about 5-10%.

Overall enterprise-buyable Twin maturity today:
Approximately 15% of where it needs to be.

The strongest current differentiator is not UI polish. It is the architecture
idea: Twin + pre-mortem + blast radius + audit + approval gates. The product
must now become much deeper and cleaner in the Twin page itself.

Design Principle Going Forward
------------------------------
Do not spam the UI.

The Twin page should not show every feature at once. It should use progressive
disclosure:

- Left side: how to start exploration.
- Center: beautiful, smooth graph canvas.
- Right side: selected-node intelligence.
- Top/status: current context and compact controls.
- Bottom/canvas overlays: only zoom/fit/legend/minimap.

Every feature must either answer one of these user questions or be hidden:

- What is this?
- Where does it live?
- What calls it?
- What does it call?
- What breaks if I change it?
- Why is it connected?
- Who owns it?
- What changed recently?
- What evidence proves this?
- Can an agent safely work here?

Comparison: Sourcegraph
-----------------------
Sourcegraph is strongest at code search, code navigation, code browsing, and
developer workflow integration.

Where Sourcegraph is ahead:

- Powerful query language.
- Symbol search/typeahead.
- Fast indexed search.
- Search contexts.
- Diff and commit search.
- Hover popovers with symbol details.
- Go to definition.
- Find references.
- Code viewer.
- Open in editor.
- Open in code host.
- Blame/history.
- Deep Search with sourced answers.
- Enterprise-grade navigation polish.

Where HelixFactory currently lacks:

- Search is basic and local to known graph nodes.
- No code viewer.
- No hover signature/docs.
- No find references.
- No go-to-definition.
- No blame/history in Twin.
- No branch/commit/diff search.
- No query grammar.
- No search result ranking comparable to Sourcegraph.
- No direct jump from graph node to source lines.

How HelixFactory can beat Sourcegraph:

- Make risk and change safety first-class.
- Add pre-mortem evidence directly to graph.
- Add blast-radius path explanation.
- Add approval/audit context to every node/change.
- Add "safe for AI?" answer Sourcegraph does not natively own.

Required features to close gap:

- Node source preview.
- Symbol detail popover.
- Find callers/callees/references.
- Search grammar: type:, path:, calls:, imports:, risk:.
- File history/owner context.
- Open code host/editor action.
- Sourced AI answer tied to graph evidence.

What Must Be Built Before It Feels "Wow"
----------------------------------------
Minimum "enterprise wow" for Twin:

1. Graph must be beautiful and calm:
   - No clutter.
   - No tiny unreadable labels everywhere.
   - Smooth hover.
   - Smooth click/recenter.
   - Strong selected state.
   - Professional color/contrast.
   - Legend and controls placed cleanly.

2. Search must feel instant:
   - Typing local.
   - Results categorized.
   - No stuck UI.
   - Explicit center/requery action.

3. Inspect panel must be valuable:
   - Code snippet.
   - Summary.
   - File/line.
   - Owner/history.
   - Callers/callees/imports.
   - Evidence.
   - Risk.

4. Impact must explain "why":
   - Not just red/yellow nodes.
   - Show dependency path.
   - Show direct vs transitive impact.
   - Show confidence/evidence.

5. Views must fit mental model:
   - Architecture overview.
   - Entry point flow.
   - Call tree.
   - Dependency graph.
   - Risk/impact map.

6. It must avoid UI spam:
   - Put advanced controls behind compact sections.
   - Show only the next useful action.
   - Use tabs in detail panel.
   - Use overlays only when active.
   - No dashboard junk inside the graph canvas.

Hard Product Standard
---------------------
The Twin page should be considered production-grade only when all of this is
true:

- No backend requests while typing.
- Graph stays smooth under expected node limits.
- Search finds symbols instantly.
- Click any node recenters cleanly.
- Right panel explains the selected node without needing another page.
- User can answer "what calls this" and "what breaks if I change this".
- Risk overlay is understandable.
- Graph does not look like a toy force-directed demo.
- Mobile does not overflow.
- E2E browser tests pass.
- Real repo demo completes under 3 minutes.

Bottom Line
-----------
HelixFactory has a differentiated thesis, but Twin is not yet buyer-grade.
It does not yet match the search depth of Sourcegraph, graph confidence of
GitNexus, map workflow polish of CodeSee, impact scoring of CodeLogic, or static
analysis depth of Understand.

The path to beating them is not to add every control to the screen. The path is
to make Twin answer high-value engineering questions faster and with evidence:

- What is this?
- What calls this?
- What will break?
- Why?
- Who owns it?
- What should we check before changing it?

If the next passes build those workflows cleanly, HelixFactory can become
stronger than a code graph product because it connects graph understanding to
risk prediction, human approval, audit, and AI governance.
