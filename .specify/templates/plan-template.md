## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **Spec Before Code**: Confirm a reviewed spec exists for this feature and the
  plan links to it.
- **Twin First**: Identify the twin data, graph traversal, or ingestion result
  that informs the implementation. If unavailable, document the graceful
  degradation path instead of proceeding as if risk is known.
- **Evidence Over Heuristics**: Any pre-mortem or risk claim MUST include file,
  line, and dependency-chain evidence from the twin.
- **Git Audit Trail**: Plan how ingestion, pre-mortem, blast radius, agent
  execution, security scan, and approval actions will be recorded as structured
  git commits.
- **Human Gates**: Confirm HIGH or CRITICAL pre-mortem findings cannot be
  auto-merged and require human approval.
- **Contextual Graph UX**: For graph work, confirm the UI is query-centered,
  selected-node-centered, or proposed-change-centered with recenter, hover
  illumination, and depth control. Full-universe graph display is prohibited.
- **Codebase-Specific Intelligence**: Any self-improvement proposal MUST be
  specific to this codebase and reject generic best-practice refinements.
- **Language Scope**: Confirm ingestion work is limited to Python, TypeScript,
  and JavaScript unless a Constitution conflict is explicitly raised.
- **Graceful Degradation**: Define at least one failure mode and how the system
  reports it without producing a false all-clear.
- **Demo Readiness**: Define how the feature can be demonstrated on a real
  public GitHub repo in under 3 minutes.
- **Working Directory Rule**: Backend commands run from `backend/`; frontend
  commands run from `frontend/`; neither runs from the project root.
- **Command Timeout Rule**: Commands exceeding 2 minutes are killed and reported
  as failed.
